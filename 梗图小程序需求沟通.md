# 木偶鱼PAF - 项目需求文档

## 一、项目概述

| 项目名称 | 木偶鱼PAF |
|----------|------------|
| 项目类型 | 微信小程序 |
| 核心功能 | 随机展示梗图，用户上传，踩/送花功能 |
| 设计原则 | 极简、傻瓜化、开源 |

---

## 二、功能需求

### 2.1 核心功能

| 功能 | 描述 |
|------|------|
| 随机展示 | 首页下拉刷新换图，已看图片不重复 |
| 送花功能 | 🌸 按钮，每人每天1次 |
| 踩功能 | 💩 按钮，每人每天3次，踩后自动换图 |
| 上传入口 | 点击右上角+上传图片，每天最多9张 |
| 审核机制 | 图片上传后需管理员审核才展示 |

### 2.2 交互规则

| 功能 | 规则 |
|------|------|
| 送花 | 每人每天1次，全局限制 |
| 踩 | 每人每天3次，不能重复踩同一张 |
| 上传 | 每人每天9张，需审核后展示 |
| 刷图 | 已看过的图片不再显示，刷完显示"上会儿班吧，球球了。" |

### 2.3 页面结构

```
┌─────────────────────────────────────┐
│ 🌸 热爱可抵漫长岁月              [+] │  ← 顶部栏
│                                     │
│                                     │
│              图片区域               │
│           (支持长图滚动)            │
│                                     │
│                                     │
│ 💩                                  │  ← 左下角踩按钮
│ 3                                   │
└─────────────────────────────────────┘
```

## 三、技术方案

### 3.1 技术选型

| 类别 | 方案 |
|------|------|
| 前端 | 微信小程序原生开发 |
| 后端 | 云开发·云函数（Node.js） |
| 数据库 | 云开发·云数据库 |
| 存储 | 云开发·云存储 |

### 3.2 审核流程

```
用户上传图片 → status: 0（待审核）
        │
        ▼
管理员审核 → status: 1（通过）/ 2（拒绝）
        │
        ▼
通过后图片进入展示池
```

## 四、数据库设计

### 4.1 images（图片表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| fileID | string | 云存储fileID |
| url | string | 图片CDN地址 |
| uploaderOpenid | string | 上传者openid |
| status | number | 0待审核/1已通过/2已拒绝 |
| dislikeCount | number | 被踩次数 |
| likeCount | number | 被送花次数 |
| date | string | 上传日期 |

| 字段 | 类型 | 说明 |
|------|------|------|

| imageId | string | 被踩图片ID |
| date | string | 日期（YYYY-MM-DD） |

### 4.3 like_logs（送花记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| openid | string | 用户openid |
| imageId | string | 被送花图片ID |
| date | string | 日期（YYYY-MM-DD） |

| 字段 | 类型 | 说明 |
|------|------|------|

## 十六、体验版发布指南

### 什么是体验版？
体验版是未发布的小程序版本，可以分享给指定用户体验测试。

### 发布步骤

1. **上传代码**
   - 微信开发者工具 → 点击「上传」按钮
   - 填写版本号和备注

2. **设置为体验版**
   - 登录 [微信公众平台](https://mp.weixin.qq.com/)
   - 管理 → 版本管理
   - 找到刚上传的开发版本
   - 点击「选为体验版本」

3. **添加体验者**
   - 版本管理 → 体验版本 → 添加体验者
   - 输入体验者的微信号
   - 体验者会收到邀请

4. **分享体验版**
   - 点击体验版本右侧的「体验二维码」
   - 下载二维码发给体验者
   - 或直接分享小程序链接

### 注意事项
- 体验版有效期：长期有效
- 体验者数量：个人主体最多15人
- 体验者需要先扫码接受邀请
