# 木偶鱼PAF - 项目需求文档

## 一、项目概述

| 项目名称 | 木偶鱼PAF |
|----------|------------|
| 项目类型 | 微信小程序 |
| 核心功能 | 随机展示梗图，用户上传，踩/送花功能 |
| 设计原则 | 极简、傻瓜化、开源 |

---

## 二、功能需求

### 2.1 核心功能

| 功能 | 描述 |
|------|------|
| 随机展示 | 首页下拉刷新换图，已看图片不重复 |
| 送花功能 | 🌸 按钮，每人每天1次 |
| 踩功能 | 💩 按钮，每人每天3次，踩后自动换图 |
| 上传入口 | 点击右上角+上传图片，每天最多9张 |
| 审核机制 | 图片上传后需管理员审核才展示 |

### 2.2 交互规则

| 功能 | 规则 |
|------|------|
| 送花 | 每人每天1次，全局限制 |
| 踩 | 每人每天3次，不能重复踩同一张 |
| 上传 | 每人每天9张，需审核后展示 |
| 刷图 | 已看过的图片不再显示，刷完显示"上会儿班吧，球球了。" |

### 2.3 页面结构

```
┌─────────────────────────────────────┐
│ 🌸 热爱可抵漫长岁月              [+] │  ← 顶部栏
│                                     │
│                                     │
│              图片区域               │
│           (支持长图滚动)            │
│                                     │
│                                     │
│ 💩                                  │  ← 左下角踩按钮
│ 3                                   │
└─────────────────────────────────────┘
```

## 三、技术方案

### 3.1 技术选型

| 类别 | 方案 |
|------|------|
| 前端 | 微信小程序原生开发 |
| 后端 | 云开发·云函数（Node.js） |
| 数据库 | 云开发·云数据库 |
| 存储 | 云开发·云存储 |

### 3.2 审核流程

```
用户上传图片 → status: 0（待审核）
        │
        ▼
管理员审核 → status: 1（通过）/ 2（拒绝）
        │
        ▼
通过后图片进入展示池
```

## 四、数据库设计

### 4.1 images（图片表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| fileID | string | 云存储fileID |
| url | string | 图片CDN地址 |
| uploaderOpenid | string | 上传者openid |
| status | number | 0待审核/1已通过/2已拒绝 |
| dislikeCount | number | 被踩次数 |
| likeCount | number | 被送花次数 |
| date | string | 上传日期 |
| createTime | number | 创建时间 |

### 4.2 dislike_logs（踩记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| openid | string | 用户openid |
| imageId | string | 被踩图片ID |
| date | string | 日期（YYYY-MM-DD） |

### 4.3 like_logs（送花记录表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| openid | string | 用户openid |
| imageId | string | 被送花图片ID |
| date | string | 日期（YYYY-MM-DD） |

### 4.4 users（用户表）

| 字段 | 类型 | 说明 |
|------|------|------|
| _id | ObjectId | 主键 |
| openid | string | 微信openid |
| role | number | 0普通/1创作者/2管理员 |

## 五、API接口设计

| 云函数 | 参数 | 说明 |
|--------|------|------|
| getRandomImage | count, seenIds | 批量获取随机图片，排除已看 |

| likeImage | imageId | 送花（云函数获取openid） |
| addImage | fileID | 添加图片（status: 0待审核） |
| admin | action, ... | 管理员操作（审核、权限验证） |

## 七、改动记录

### 2024-XX-XX 功能完善

#### 1. 布局优化
- 图片居中显示，使用 `min-height: 100vh` + `justify-content: center`
- 长图支持滚动查看
- 踩按钮移至左下角，避免右手误触

- 每人每天只能送1次花
- 本地存储记录送花日期

#### 3. 上传限制
- `addImage` 云函数增加每日上传数量检查
- 每人每天最多上传9张图片

#### 4. 审核功能完善
- 上传后默认 `status: 0`（待审核）
- 新增 `pages/admin/admin` 审核页面
- 审核页面三个Tab：待审核/已通过/已拒绝
- 管理员权限验证（ADMIN_OPENIDS 配置）

#### 5. 已看图片不重复
- `getRandomImage` 云函数支持 `seenIds` 参数
- 前端记录已看图片ID列表
- 图片刷完显示提示文字

#### 6. 隐藏入口
- 连续点击5次"热爱可抵漫长岁月"进入审核页面
- 非管理员可进入审核页面但无法操作
- 云函数验证管理员权限

#### 7. UI细节
- 鲜花按钮显示"1"（每天次数）
- 踩按钮显示"3"（每天次数）
- 顶部显示"热爱可抵漫长岁月"
