# 木偶鱼小程序更新记录

## 2026-03-25 更新

### 1. 升级 Web SDK 版本并修复数据库访问问题
- **文件**：`admin.html`
- **改动**：
  - 将 CDN 链接更新为官方推荐地址：`https://static.cloudbase.net/cloudbase-js-sdk/latest/cloudbase.full.js`
  - 使用 `cloudbase` 全局变量进行初始化
  - 添加匿名登录逻辑，在访问数据库前先进行身份认证
  - 添加详细的错误处理和用户提示，提供重试按钮
- **原因**：解决 SDK 加载失败、`tcb is not defined` 和 `Cannot read properties of null (reading 'scope')` 错误
- **效果**：Web 管理后台现在可以正常访问数据库和存储资源，解决了数据库访问权限问题

### 2. 创建云函数获取临时 URL
- **文件**：`cloudfunctions/getTempUrls/`
- **改动**：
  - 创建云函数 `getTempUrls` 用于获取云存储文件的临时访问链接
  - 修改 `admin.html` 使用云函数获取临时 URL
  - 添加 `transformCloudUrl` 函数作为备用方案，将 `cloud://` URL 转换为 `https://` URL
- **原因**：Web 端直接调用 `getTempFileURL` 返回 undefined，需要通过云函数获取
- **效果**：Web 管理工具可以正常显示图片，解决了图片加载问题

### 3. 配置云函数安全规则
- **设置**：
  - 在云开发控制台配置 `getTempUrls` 云函数的安全规则
  - 允许未登录用户调用云函数：`{"*": {"invoke": true}}`
- **原因**：默认情况下云函数不允许未登录用户调用
- **效果**：Web 管理工具可以正常调用云函数获取临时 URL

### 4. 添加图片预览模态框
- **文件**：`admin.html`
- **改动**：
  - 添加图片预览模态框，点击图片在模态框中显示大图
  - 不再直接打开新窗口下载图片
  - 支持点击任意位置关闭预览
- **效果**：用户体验更好，预览图片更方便

### 2. 修复 iPhone SE 3 图片加载问题
- **文件**：`cloudfunctions/getRandomImage/index.js`
- **改动**：修改云函数，使用 `cloud.getTempFileURL` 获取临时 URL 替代原来的 cloud:// URL
- **原因**：某些 iOS 设备对 cloud:// URL 支持不佳
- **效果**：所有 iOS 设备（包括 iPhone SE 3）现在都能正常加载图片

## 如何使用 Web 管理工具

### 前置条件
1. **云开发控制台配置**（必须）：
   - 进入云开发控制台 → 环境设置 → 安全配置
   - ✅ 开启「未登录用户访问云资源权限」
   - ✅ 开启「允许匿名登入」
   - 数据库权限设置：images 集合设置为「所有用户可读，创建者可写」
   - 存储权限设置：设置为「所有用户可读，创建者可写」

### 启动步骤
1. 打开终端，进入项目目录
2. 运行以下命令之一启动本地服务器：
   - Python 方式：`python -m http.server 9000`
   - 或使用 VS Code Live Server 插件
3. 在浏览器中访问 `http://localhost:9000/admin.html`
4. 输入开发者 OpenID 登录（开发者 OpenID：oWatD3bwu0aVaFTPvrrerAU_C2zY）
5. 即可使用批量上传、审核、删除等功能

### 常见问题
- **如果遇到"初始化失败"错误**：请检查云开发控制台的权限设置是否正确
- **如果图片无法显示**：请检查数据库和存储的权限设置
- **如果上传失败**：请检查存储权限设置

## 注意事项
- Web 管理工具需要本地 HTTP 服务器运行，不能直接打开 HTML 文件
- 确保云开发环境的权限设置正确，否则可能无法访问数据
- 上传图片时会自动检测重复，避免重复上传
- 开发者拥有最高权限，可无限制上传图片
