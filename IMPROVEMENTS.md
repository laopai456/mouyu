# 项目改进建议

> 最后更新：2026-03-31

---

## 背景

本项目采用微信小程序+云开发的快速开发模式。当前功能完整，但代码质量和可维护性有提升空间。本文档记录已知问题和改进建议，不影响当前运营。

---

## 现状评估

| 维度 | 评分 | 说明 |
|------|------|------|
| 功能完整性 | ⭐⭐⭐⭐ | 该有的都有 |
| 代码质量 | ⭐⭐ | 缺少规范，过度耦合 |
| 可维护性 | ⭐⭐ | 改一处怕动全身 |
| 可扩展性 | ⭐⭐ | 加功能要从零搭 |

---

## 待改进项

### 1. API 封装

**问题**：页面直接调 wx.cloud.callFunction，耦合高

**改进**：
```
├── api/
│   ├── image.js      # 图片相关 API
│   ├── user.js       # 用户相关 API
│   └── request.js    # 请求基类封装
```

---

### 2. 组件拆分

**问题**：index.js 很长，功能堆一起

**改进**：
```
├── components/
│   ├── ImageCard/     # 图片卡片组件
│   ├── ImageGrid/     # 瀑布流组件
│   ├── ImagePreview/  # 预览组件
│   └── Effects/       # 特效组件
```

---

### 3. 状态管理

**问题**：用 Page data 硬撑，跨页面共享麻烦

**改进**：引入 Pinia（小程序可用）或使用 wx.store 方案

---

### 4. 异常处理

**问题**：网络错误只弹 toast，没重试

**改进**：
- 请求封装增加自动重试（3次）
- 超时统一处理
- 加载状态统一管理

---

### 5. 样式规范

**问题**：颜色、间距随意，wxss 散落

**改进**：
```
├── styles/
│   ├── variables.wxss   # 颜色、字体变量
│   ├── mixins.wxss       # 复用样式
│   └── common.wxss       # 通用样式
```

---

### 6. 工具函数

**问题**：没有统一的工具目录

**改进**：
```
├── utils/
│   ├── md5.js          # MD5 计算
│   ├── format.js       # 格式化函数
│   └── storage.js      # storage 封装
```

---

### 7. 接口文档

**问题**：没有接口文档，云函数改了不知道要同步

**改进**：创建 API.md，记录所有云函数的参数和返回值

---

## 优先级

| 优先级 | 改动 | 收益 |
|--------|------|------|
| 低 | 样式规范 | 代码整洁 |
| 低 | 组件拆分 | 便于复用 |
| 中 | API 封装 | 便于维护 |
| 中 | 异常处理 | 稳定性 |
| 高 | 接口文档 | 团队协作 |

---

## 说明

- 以上改进项为长期规划，不影响当前功能
- 建议在新增功能时顺便改进，而不是大规模重构
- 小程序代码包有限制（2MB），组件化需控制体积
